# 第十阶段 Smoke Checklist

这份清单只覆盖当前主干最关键闭环，目标是给后续大功能分支提供一个可快速回归的稳定基线。
第十一阶段继续沿用这份清单，并在末尾追加 Markdown 快捷输入的验证项。
第十二阶段继续沿用这份清单，并在末尾追加 LaTeX 公式 MVP 的验证项。
第十三阶段继续沿用这份清单，并在末尾追加正文图片、本地资源与 notebook 封面 MVP 的验证项。
第十六阶段继续沿用这份清单，并按 notebooks 首页 / 内部工作区的新 UI 更新相关步骤。

## 使用前准备

1. 运行 `npm run tauri dev`。
2. 确认应用能正常启动到主界面。
3. 若要验证恢复流程，先在设置页准备至少 1 份可恢复备份。

## 1. 创建笔记本

1. 进入“笔记本”。
2. 在首页点击 `新建笔记本`，输入 `阶段十测试本`。
3. 预期：首页封面网格出现新笔记本卡片。

## 2. 创建文件夹

1. 单击 `阶段十测试本` 卡片进入笔记本内部工作区。
2. 点击左栏 `新建文件夹`。
3. 预期：左树出现默认命名的新文件夹，双击名称后可改成 `收集箱`。

## 3. 创建文件

1. 选中 `收集箱`。
2. 点击左栏 `新建文件`。
3. 预期：左树出现默认命名的新文件，双击名称后可改成 `阶段十测试文件`；中间区域进入正文编辑面板。

## 4. 编辑正文并自动保存

1. 在正文中输入一段文字，例如 `这是阶段十自动保存测试。`。
2. 停止输入，等待超过 800ms。
3. 预期：保存状态依次经过“编辑中 / 待保存”“保存中”“已保存”，最近更新时间刷新。
4. 切到其他文件再切回当前文件。
5. 预期：刚才输入的正文仍然存在。

## 5. 未保存时切换 section 被拦截或先 flush

1. 在当前文件继续输入一小段新内容。
2. 不等待 800ms，立即点击左侧“复习日历”或“设置”。
3. 预期：不会静默丢稿；应先触发 flush，保存成功后才切走。
4. 若保存失败，预期：仍停留在笔记本工作区，并出现明确中文错误提示。

## 6. 搜索命中文件

1. 回到笔记本首页顶部搜索框，或任意其他 section 顶部搜索框。
2. 搜索 `阶段十自动保存测试` 或文件标题。
3. 点击搜索结果。
4. 预期：自动打开目标 note，定位到对应笔记本与文件；若从笔记本首页搜索进入，应看到正文首次命中高亮约 5 秒。

## 7. 标签添加 / 移除

1. 在当前文件的标签区添加标签 `重点`。
2. 预期：标签立即出现在当前标签列表。
3. 点击移除 `重点`。
4. 预期：标签从当前文件中消失，无重复关系残留。

## 8. 绑定复习方案

1. 进入“复习日历”。
2. 创建方案 `阶段十方案`，步骤填写 `0`、`3`。
3. 回到“笔记本”，打开 `阶段十测试文件`。
4. 在复习计划区绑定 `阶段十方案`，起始日期保持当天。
5. 预期：显示当前绑定方案和步骤预览。

## 9. 日历任务点击打开 note

1. 回到“复习日历”。
2. 找到当月任务列表中的 `阶段十测试文件`。
3. 点击任务打开对应笔记。
4. 预期：应用跳回笔记本工作区并打开目标 note。

## 10. 标记完成 / 取消完成

1. 在复习日历中找到同一条任务。
2. 点击“标记完成”。
3. 预期：任务状态变为已完成。
4. 再点击取消完成。
5. 预期：任务恢复为未完成，重复 3 至 5 次应稳定成功。

## 11. 手动备份

1. 进入“设置”。
2. 点击“立即创建备份”。
3. 预期：提示创建成功，备份列表新增一条可恢复备份。
4. 若首次打开“设置”，预期：备份列表会先显示轻量基础信息，不会因为全量严格校验而明显卡顿。

## 12. 恢复备份

1. 在笔记本中新建一段容易识别的临时内容。
2. 进入“设置”，选择刚才创建的备份并确认恢复。
3. 预期：恢复成功后应用自动 reload。
4. reload 完成后检查临时内容是否按备份状态回退。
5. 若恢复失败，预期：页面会显示明确中文原因，例如压缩包损坏、清单无效、数据库版本不兼容等，而不是统一“恢复失败”。

## 13. 手动重建搜索索引

1. 进入“设置”。
2. 点击“手动重建搜索索引”。
3. 预期：显示重建中提示，完成后提示成功。
4. 再次搜索现有文件标题或正文。
5. 预期：搜索结果正常返回。

## 14. 窗口关闭前未保存保护

1. 打开一个 note，输入一小段新内容。
2. 不等待自动保存，直接关闭主窗口。
3. 预期：不会静默丢最后一段内容；应用会先拦截并尝试 flush，保存成功后才允许关闭。
4. 若保存失败，预期：窗口保持打开，并显示“当前笔记仍有未保存内容，已阻止关闭或刷新。请先等待保存完成，或复制内容后再操作。”

## 15. 刷新前未保存保护

1. 打开一个 note，输入一小段新内容。
2. 不等待自动保存，直接触发 WebView reload / 浏览器刷新。
3. 预期：出现系统原生离开提示，不允许静默丢最后一段内容。
4. 若取消离开，预期：仍停留在当前页面，可继续编辑或等待保存完成。

## 16. Markdown 快捷输入转换

1. 回到 `阶段十测试文件` 或任意可编辑 note。
2. 在空段落输入 `# `。
3. 预期：当前行立即转换为一级标题。
4. 新起一行，在空段落输入 `## `。
5. 预期：当前行立即转换为二级标题。
6. 再新起一行，分别输入 `- ` 和 `* `。
7. 预期：两种写法都会转换为无序列表。
8. 再新起一行，输入 `1. `。
9. 预期：转换为从 `1` 开始的有序列表。

## 17. Markdown 快捷输入后的回车与持久化

1. 在空列表项上直接按回车。
2. 预期：退出列表并回到普通段落。
3. 在标题行末尾按回车。
4. 预期：下一行是普通段落，不继续保持标题。
5. 使用上述任一快捷输入后等待超过 800ms。
6. 预期：自动保存仍正常完成，保存状态依次更新。
7. 切到其他 note 或其他 section 再返回。
8. 预期：标题与列表结构仍在，flush 链路未被破坏。
9. 搜索刚才输入的标题文字或列表正文。
10. 预期：搜索仍能命中对应 note；标签、复习、备份恢复链路无异常。

## 18. 行内公式插入、编辑与持久化

1. 回到任意可编辑 note，点击工具栏“行内公式”。
2. 输入 `E=mc^2` 并确认插入。
3. 预期：正文中立即渲染出行内公式，且与普通文本保持同一行展示。
4. 等待超过 800ms，确认保存状态正常变为“已保存”。
5. 双击该公式，修改为 `E=mc^3` 并确认更新。
6. 预期：公式立即按新源码重新渲染。
7. 切到其他 note 或其他 section，再回到当前 note。
8. 预期：行内公式仍能正确显示，源码修改结果被保留。

## 19. 块级公式插入、编辑与持久化

1. 在正文中点击工具栏“块级公式”。
2. 输入 `\\frac{a}{b}` 并确认插入。
3. 预期：正文中出现独立块级公式，与段落有明确视觉区分。
4. 双击该块级公式，修改为 `\\frac{a+b}{c}` 并确认更新。
5. 预期：块级公式立即重新渲染。
6. 等待自动保存完成后，切换 note、切换 section，或重开该 note。
7. 预期：块级公式仍能正确显示，保存链路与 flush 链路不受影响。

## 20. 公式搜索与非法公式容错

1. 在全局搜索中搜索 `mc^3` 或 `frac`。
2. 预期：能命中刚才包含公式源码的 note。
3. 点击搜索结果打开目标 note。
4. 预期：打开后公式仍正常显示。
5. 再次点击“行内公式”或“块级公式”，输入非法源码，例如 `\\frac{`。
6. 预期：对话框内出现明确中文错误提示，不写入正文，不导致 note 崩坏。
7. 关闭对话框后继续编辑普通正文，等待自动保存完成。
8. 预期：标签、复习、备份恢复、未保存保护等既有链路无异常。

## 21. 正文图片插入、自动保存与重开恢复

1. 回到任意可编辑 note，点击工具栏“图片”。
2. 在系统文件选择器中选择一张本地 `png`、`jpg`、`jpeg`、`webp` 或 `gif` 图片。
3. 预期：图片被插入到当前光标位置并立即显示；不会依赖用户原始绝对路径。
4. 等待超过 800ms。
5. 预期：保存状态正常变为“已保存”。
6. 切到其他 note，再切回当前 note。
7. 预期：图片仍正常显示。
8. 关闭并重新启动应用，重新打开当前 note。
9. 预期：图片仍能正常显示，正文其它部分继续可编辑。

## 22. 图片-only note 不被清空

1. 创建一个新 note，不输入普通文字，只插入一张图片。
2. 等待自动保存完成。
3. 切到其他 note，再切回该 note。
4. 预期：该 note 不会因为正文文本为空而被归一化成空内容，图片仍保留。

## 23. Notebook 封面设置、替换与清除

1. 回到笔记本首页，对某个 notebook 卡片执行右键。
2. 点击“更换封面”，选择一张本地图片。
3. 预期：首页封面网格中的该卡片立即显示新封面。
4. 再次右键同一卡片，选择另一张图片。
5. 预期：卡片封面更新为新图。
6. 再次右键该卡片，点击“清除封面”。
7. 预期：卡片回到纯色占位封面。
8. 刷新应用或重启后再次查看。
9. 预期：设置、替换、清除后的状态均能正确保留。

## 24. 备份恢复中的图片与封面 roundtrip

1. 在某个 note 中插入正文图片，并为对应 notebook 设置封面。
2. 进入“设置”，创建一份新备份。
3. 回到应用中修改当前数据，例如删除图片 note 内容或更换/清除封面。
4. 再进入“设置”，恢复刚才创建的备份。
5. 预期：恢复完成并 reload 后，正文图片与 notebook 封面都回到备份时的状态，资源路径不失效。

## 25. 图片与资源错误容错

1. 验证一次“图片”按钮后直接取消文件选择。
2. 预期：视为正常取消，不出现致命错误，不污染当前正文。

## 26. 笔记本首页与内部态切换

1. 进入“笔记本”。
2. 预期：默认先看到笔记本首页，而不是旧三栏工作区。
3. 单击任意 notebook 卡片。
4. 预期：进入沉浸式内部工作区，左侧主导航 Rail 隐藏，顶部左上角显示返回箭头和当前笔记本名称。
5. 点击返回箭头。
6. 预期：回到笔记本首页，Rail 再次显示。

## 27. 右侧功能区折叠与记忆

1. 进入某个 notebook，打开任意 note。
2. 点击右侧功能区顶部的折叠按钮。
3. 预期：右侧收起为窄竖条，中间编辑区变宽。
4. 再点击竖条上的展开按钮。
5. 预期：标签和复习计划区恢复显示，内容正常可用。
6. 再次折叠后离开该 notebook，再重新进入。
7. 预期：右侧保持上次折叠 / 展开的状态。

## 28. Command + Delete 删除确认

1. 在笔记本首页选中一个 notebook 卡片后按 `Command + Delete`。
2. 预期：弹出“确定删除这个笔记本吗”的确认框，不会静默删除。
3. 进入某个 notebook，选中左树中的文件夹或文件后按 `Command + Delete`。
4. 预期：分别弹出“确定删除这个文件夹吗”或“确定删除这个文件吗”的确认框。
5. 把焦点放到正文编辑器或任意输入框后再按 `Command + Delete`。
6. 预期：不触发 notebooks 删除确认。

## 29. 左树双击重命名

1. 在 notebooks 内部左树中双击某个文件夹名称。
2. 预期：进入重命名输入态。
3. 修改名称后按 `Enter`。
4. 预期：出现小型确认态，点击“确认”后名称保存。
5. 对某个文件重复相同步骤。
6. 预期：文件重命名流程一致；按 `Esc` 或“取消”时不会落库。
3. 若可人工模拟封面保存失败场景，验证“新封面导入成功但数据库更新失败”。
4. 预期：不会留下新的孤儿封面资源文件；界面提示“笔记本封面保存失败，请稍后重试。”
5. 若可人工模拟资源缺失场景，例如删除某个 `resources/images/...` 文件后再打开 note。
6. 预期：对应图片节点显示降级占位与“图片资源不存在或已损坏。”提示，但整篇 note 不崩坏，搜索、标签、复习、备份列表、未保存保护仍可继续使用。
