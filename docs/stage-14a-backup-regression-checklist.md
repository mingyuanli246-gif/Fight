# 第十四A阶段 Backup Regression Checklist

这份清单只覆盖第十四A阶段收口的备份链路优化与相关保护，不替代现有主 smoke checklist。

## 1. 手动备份：空资源

步骤：
1. 启动应用，确保当前数据目录下没有正文图片和 notebook 封面。
2. 进入“设置”，点击“立即创建备份”。

预期：
1. 能正常创建备份，不报错。
2. 新备份项会立即出现在列表顶部。
3. 后台补刷新完成后，列表无重复项，顺序稳定。
4. 控制台能看到 `[backup.perf]` 分阶段日志。

## 2. 手动备份：小资源

步骤：
1. 在某个 note 中插入 1 张小图片，并为当前 notebook 设置 1 张封面。
2. 进入“设置”，点击“立即创建备份”。

预期：
1. 创建成功提示正常出现。
2. 新备份无需等待全量重扫即可立即显示。
3. 后台补刷新后，该备份仍只出现 1 条。

## 3. 手动备份：多资源

步骤：
1. 准备多个包含正文图片和封面的 notebook / note。
2. 进入“设置”，点击“立即创建备份”。

预期：
1. 备份能够成功完成。
2. 列表立即插入新项。
3. 后台刷新后无重复、无顺序错乱。
4. 控制台日志能看到数据库快照、资源打包、zip 完成、结果构造等阶段。

## 4. 图片 note + notebook 封面 + 备份恢复 roundtrip

步骤：
1. 在某个 note 中插入正文图片，并为对应 notebook 设置封面。
2. 创建一份新备份。
3. 修改当前数据，例如删除图片内容或清除封面。
4. 恢复刚才创建的备份。

预期：
1. restore 前会继续走严格校验。
2. reload 后，正文图片和 notebook 封面都回到备份时状态。
3. 资源路径不失效，图片和封面都能正常显示。

## 5. 从 notebooks 切到 settings 后再手动备份

步骤：
1. 在 notebooks 中打开一个 note，输入一段新内容。
2. 不等待自动保存，直接切到“设置”。
3. 到达“设置”后立刻点击“立即创建备份”。

预期：
1. 切换 section 前会先走 flush 保护，不会静默丢稿。
2. 进入设置后可正常创建备份。
3. 备份列表仍按“立即插入 + 后台补刷新”工作。

## 6. 恢复备份前 dirty note 阻断

步骤：
1. 打开一个 note，制造未保存内容。
2. 切到“设置”，选择某份备份并点击恢复。

预期：
1. 恢复前仍会调用 notebooks flush guard。
2. 若保存失败，应阻止 restore，并显示明确中文提示。
3. 备份优化不应绕过这条保护链路。

## 7. 设置页首次打开时备份列表正常

步骤：
1. 准备多份现有备份。
2. 打开应用并进入“设置”。

预期：
1. 首次打开设置页时仍会从真实目录读取备份列表。
2. 列表内容、排序与可恢复状态正常。
3. 控制台能看到 `reason=initial-load` 的刷新 perf 日志。

## 8. 创建备份后列表立即可见且最终刷新正确

步骤：
1. 保持“设置”页打开。
2. 连续创建 1 到 2 次备份，观察列表变化。

预期：
1. 每次创建成功后，新项都会立即出现在列表顶部。
2. 后台刷新完成后，不应出现重复项、顺序闪烁或旧请求覆盖新结果。
3. 控制台能看到 `reason=post-create` 的刷新 perf 日志。
